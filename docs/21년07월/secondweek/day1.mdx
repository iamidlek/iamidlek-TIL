## Quiz
#### 문제 1. 변수 name에 kim값이 있을때 출력 결과가 다른 것은?
1. 'Hello, ' + name + '!'
2. 'Hello, kim !'
3. &#96;Hello, ${name}!&#96;	
4. 'Hello, kim!'

<details>
  <summary>
  답안 확인
  </summary>
  <ul class='quiz'>
    <li>②</li>
    <li>kim 과 ! 사이의 공백은 다른 선택지에서는 나타나지 않는다.</li>
  </ul>
</details>

#### 문제 2. 배열 내장함수의 설명으로 틀린 것은?
1. map을 사용하려면 빈 배열 선언이 선행되어야 한다
2. indexOf는 찾는 값이 없을 경우 -1을 retrun한다
3. filter에는 조건을 검사하는 함수를 넣는다
4. slice는 원래의 배열을 건드리지 않는다

<details>
  <summary>
  답안 확인
  </summary>
  <ul class='quiz'>
    <li>①</li>
    <li>map 의 파라미터로 변화함수를 줄 수 있고</li>
    <li>각 들어오는 값들을 변화함수에 넣고 반환한다</li>
  </ul>
</details>


#### 문제 3. 클래스의 설명으로 틀린 것은?
1. JavaScript의 ES6부터 추가되었다
2. 객체 생성자로 구현했던 코드를 명확하고 깔끔하게 해준다
3. 메서드를 만들면 자동으로 prototype 등록이 된다 
4. 다른 클래스의 상속이 어려워졌다

<details>
  <summary>
  답안 확인
  </summary>
  <ul class='quiz'>
    <li>④</li>
    <li>클래스를 사용함으로 다른 클래스를 쉽게 상속 받는다</li>
    <li>extends 키워드를 사용, super()함수가 상속받은 클래스의 생성자를 가르킨다</li>
  </ul>
</details>
